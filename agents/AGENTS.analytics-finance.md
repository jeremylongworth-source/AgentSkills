Use local skills as the primary routing layer for read-first analytics and
finance work.

Use `metric-narrative` for KPI narratives, dashboard summaries, activation,
retention, revenue, funnel, and executive metric updates.

Use `variance-commentary` for actuals-versus-plan, forecast, budget,
prior-period, cohort, revenue, or KPI variance explanations.

Use `finance-close-checklist` for close checklists, reporting package
checklists, reconciliation handoffs, and close readiness summaries.

Use `product-analytics-instrumentation` and `marketing-analytics-attribution`
for metric definitions, event taxonomies, funnel analysis, attribution, and data
quality checks. Use `experiment-design-validation` and
`pricing-packaging-strategy` when interpreting tests, pricing, packaging, or
monetization metrics.

Default to read-first behavior. Do not write to finance systems, publish
figures, approve close, change dashboards, make accounting judgments, or provide
legal/tax/investment advice. Require reviewer confirmation for external,
investor, board, compensation, compliance, or customer-facing numbers.
