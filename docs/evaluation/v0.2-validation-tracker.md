## Packet Workflow

1. Collect a real-input packet with `real-input-packet-template.md`.
2. Run a baseline with no target bundle or with the closest prior bundle.
3. Run the same prompt with the target skillset.
4. Score both outputs with `skill-quality-rubric.md`.
5. Record pass, revise, block, merge, split, defer, or retire.
6. Patch only concrete routing, output, trigger, freshness, or safety gaps.
7. Re-run `scripts/release-check.ps1` after each completed bundle group.

## Promotion Rules

- Keep: real-input packets show useful artifacts, acceptable edit burden, and
  safe approval gates.
- Revise: repeated missing section, noisy routing, stale guidance, or weak
  safety behavior appears.
- Merge or split: real-input work shows overlapping skills or overloaded
  bundle scope.
- Defer: the bundle needs private connectors, legal/security review, or domain
  evidence before it can be useful.
- Retire: the bundle adds overhead without improving the artifact.
