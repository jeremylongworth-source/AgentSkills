# v0.2.0 Release Notes

This release expands AgentSkills from an initial builder-focused library into a
larger prerelease portfolio of scenario-tested alpha skillsets.

Recommended GitHub release state: prerelease

Reason: v0.2 adds a large alpha bundle portfolio. Validation, install checks,
CI configuration, and publish dry-run are green locally, but several new
business and creator bundles still need real or anonymized input packets before
they should be described as proven workflows.

## Validation Improvements

- `scripts/validate-skill-files.py` provides repo-local skill structure checks
  and lets validation run even when the Codex skill-creator validator is not
  installed.
- `scripts/validate-skillsets.py` now checks that `all` includes every skill.
- Expanded business bundles must have matching bundle briefs with governance
  sections.
- `scripts/validate-scenarios.py` checks route scenarios against
  `tests/expected-routing.yaml` and real skill folders.
- `scripts/validate-docs.py` checks README and bundle brief index consistency.
- `scripts/release-check.ps1` now dry-runs every skillset install.
- `scripts/release-check.ps1` now runs a fresh Codex home smoke test for
  `game-dev` and `all`.
- GitHub Actions runs the release gate on push, pull request, and manual runs
  with publish dry-run skipped in CI.
- `scripts/validate-docs.py` now checks validation counts, public repo docs,
  evaluation packet/report links, and release-gate language.

## Safety and Governance

- New business bundles default to draft-first or read-first behavior.
- Customer sends, outbound sends, billing/account changes, production actions,
  public reporting, and other high-risk actions require approval.
- Analytics and finance workflows explicitly avoid accounting, tax, legal,
  audit, or investment advice.

## Release Gate

Recommended release state: prerelease until real-input packet coverage expands.

Before publishing:

```powershell
.\scripts\release-check.ps1
```

Expected result:

- skill validation passes
- skillset validation passes
- scenario validation passes
- docs consistency validation passes
- every skillset install dry-run passes
- fresh Codex home install smoke passes
- secret and local-path scans pass
- `gh skill publish --dry-run` returns `ok`
