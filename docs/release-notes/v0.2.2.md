# v0.2.2 Release Notes

This patch release adds the focused project-onboarding skill needed to set up
AgentSkills in a new repository without installing overly broad bundles. It
also adds a customer-success health review skill and updates support/revenue
routing around the first-five-hires operating model.

Recommended GitHub release state: normal release

Reason: v0.2.2 adds small, scenario-tested package functionality on top of the
stable v0.2.1 distribution path. Existing role bundles remain alpha unless
separately promoted with real-input evidence.

## What Changed

- Added `agentskills-project-onboarding` for focused AgentSkills setup in new
  and existing repositories.
- Added a forward-test report and before/after example for project onboarding.
- Added `customer-health-review` for account health, renewal risk, QBR,
  adoption, and success-plan workflows.
- Added support and revenue routing scenarios for customer health and pipeline
  review.
- Updated `all`, `support-success`, and `revenue-growth` skillsets.
- Added a lean-company operating model note for the first-five-hires expansion
  lens.

## Validation

Before publishing this release, local validation passed with:

- 166 skill files
- 33 skillsets
- 123 routing scenarios
- docs consistency validation
- fresh Codex home install smoke for `game-dev` and `all`
- `gh skill publish --dry-run`

## Maturity Boundary

This release keeps the distribution path stable, but it does not promote all
business workflows beyond alpha. Customer success, revenue, owner-operator, and
other business bundles still need reviewer packets or real/anonymized operating
evidence before they should be described as proven workflows.
