# v0.2 Release Handoff

Status date: 2026-04-30

Use this page as the maintainer handoff for the expanded v0.2 release
candidate.

## Current Status

Local release gate: passed

Recommended release state: prerelease

Reason: v0.2 adds a large scenario-tested alpha portfolio. Repo validation,
install checks, fresh-home smoke tests, docs checks, secret/local-path scans,
and publish dry-run pass locally. Several new business, creator, executive,
data, and engineering bundles still need real or anonymized input packets before
they should be promoted beyond alpha or described as proven workflows.

Next normal-release path: use `docs/stable-release-plan.md` to separate stable
distribution readiness from alpha bundle maturity.

## Release Owner Actions

1. Review the final diff and make sure no private packets, logs, generated
   caches, secrets, or local machine paths are included.
2. Push the intended release branch and wait for GitHub Actions validation.
3. Review `docs/release-candidate-v0.2.md`,
   `docs/evaluation/v0.2-validation-tracker.md`, and
   `docs/release-notes/v0.2.0.md`.
4. Publish v0.2 as a prerelease unless the release owner explicitly accepts the
   remaining alpha evidence gaps.
5. After tagging, verify the GitHub release page and install path.

## Normal Release Path

For a future non-prerelease patch, first prove the distribution path:

- Windows, macOS, and Linux release gates pass in GitHub Actions.
- Local Windows release gate passes with publish dry-run.
- Published-tag install smoke test passes.
- Release notes state that role bundles remain alpha unless separately
  promoted.
- Evidence gaps stay listed in the validation tracker.

## Do Not Claim Yet

- Do not claim cross-platform proof until Windows, macOS, and Linux release
  checks have run.
- Do not claim business, creator, executive, data, finance, security, or legal
  workflow proof beyond alpha until real-input packet coverage exists.
- Do not claim remote CI passed until the GitHub Actions run completes.
- Do not treat scenario validation as real-world adoption evidence.
