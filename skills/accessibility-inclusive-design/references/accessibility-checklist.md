# Accessibility Checklist

## Structure And Navigation

- Page or screen has meaningful title, headings, landmarks, and reading order.
- Keyboard navigation reaches all interactive elements.
- Focus is visible, logical, and not trapped unless intentionally inside a modal/dialog.
- Links and buttons are distinguishable by role and accessible name.

## Interaction

- Forms have labels, instructions, errors, and recovery paths.
- Dynamic status changes are announced where needed.
- Custom controls follow expected keyboard and screen-reader patterns.
- Pointer gestures have alternatives where required.
- Time limits, animations, and interruptions are controllable where relevant.

## Visual And Media

- Text and important UI meet contrast expectations.
- Layout supports zoom, reflow, and responsive use without loss of content.
- Color is not the only signal.
- Captions, transcripts, and alternatives exist for meaningful audio/video.
- Motion, flashing, and camera effects are reviewed for safety and comfort.

## Verification

- Automated checks are run where tooling exists.
- Manual keyboard pass is completed.
- Screen-reader or assistive-tech behavior is checked for high-risk flows.
- Findings are prioritized by task impact and severity.
