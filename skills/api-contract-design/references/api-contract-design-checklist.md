# API Contract Design Checklist

## Contract Fields

- Consumers
- Use cases
- Resources
- Methods or endpoints
- Auth and permissions
- Request schema
- Response schema
- Validation rules
- Error model
- Status codes
- Pagination/filtering/sorting
- Idempotency
- Versioning
- Compatibility
- Examples

## Review Questions

- Can a client implement against this contract without reading server code?
- Are required and optional fields clear?
- Are errors predictable and machine-readable?
- Are sensitive fields excluded?
- Are breaking changes identified?
- Is the contract testable?
