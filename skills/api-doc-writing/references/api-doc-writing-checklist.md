# API Doc Writing Checklist

## Operation Template

- Name:
- Purpose:
- Method or function:
- Path or signature:
- Auth:
- Parameters:
- Request body:
- Response:
- Errors:
- Rate limits:
- Side effects:
- Idempotency:
- Example:
- Source of truth:

## Review Checks

- Examples compile or are syntactically valid
- Required versus optional fields are clear
- Error cases are covered
- Auth scopes are not over-promised
- Internal-only details are excluded
- Version and deprecation notes are present when needed
