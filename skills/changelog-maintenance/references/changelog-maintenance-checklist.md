# Changelog Maintenance Checklist

## Categories

- Added
- Changed
- Deprecated
- Removed
- Fixed
- Security
- Docs
- Validation

## Entry Rules

- One change per bullet
- User-visible impact first
- Internal detail only when relevant
- Version/date accurate or marked TBD
- Breaking changes explicit
- Links checked when included

## Release Prep

- Confirm target version
- Move Unreleased entries
- Add date or TBD
- Compare against git diff or PR list
- Verify validation output if mentioned
