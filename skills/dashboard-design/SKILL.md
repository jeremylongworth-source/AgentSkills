# Dashboard Design

## Core Workflow

1. Identify dashboard audience, decisions supported, refresh cadence, and
   source systems.
2. Select metrics that answer the decision, not every available number.
3. Define layout, sections, charts, filters, segments, drilldowns, thresholds,
   and annotations.
4. Include metric definitions, caveats, data quality indicators, and owner.
5. Specify actions a viewer should take from each section.
6. Add validation and review steps before publication.

## Safety Rules

- Do not invent numbers, charts, metric definitions, or source systems.
- Do not publish executive, financial, customer, or compliance dashboards
  without source and reviewer confirmation.
- Do not use dashboards to imply causation without evidence.

## Deliverable Shape

For dashboard specs, provide:

- Audience and decision purpose
- Metric list and definitions
- Layout and chart plan
- Filters and segments
- Drilldowns and annotations
- Data quality and caveats
- Action rules
- Review and refresh cadence
