# Guided Interview Checklist

## Before Asking

- Target artifact is clear.
- Existing source material has been used before asking new questions.
- The first question round has a stated purpose.
- Questions are ordered by decision value, not by template order.

## Question Quality

- Ask one to three questions per round.
- Each question changes scope, requirements, acceptance criteria, risk,
  ownership, evidence, or approval needs.
- Questions are specific enough for the user to answer without guessing what
  the agent means.
- Offer defaults only when they are clearly labeled as assumptions.
- Avoid asking for information that can be inferred from supplied artifacts.

## Alignment

- Summarize material decisions after important answers.
- Keep confirmed facts separate from assumptions and open questions.
- Preserve stakeholder language when wording affects buy-in.
- Make tradeoffs visible when the user chooses scope, priority, timing, or
  quality constraints.

## Stop Conditions

- Enough information exists to create a useful draft.
- Remaining gaps can be listed as assumptions or open questions.
- Additional questions would be low-value or premature.
- The user asks to stop, switch modes, or produce the artifact.

## Handoff

- Final artifact names remaining gaps.
- Decisions and assumptions are traceable.
- Approval needs are visible.
- The next recommended action is concrete: review, answer open questions,
  validate evidence, create BRD, create PRD, create implementation plan, or
  schedule stakeholder review.
