# README Upgrade Checklist

## Core Sections

- Project name and one-line purpose
- Why it exists
- Quick install
- First successful use
- Available modules, packages, or skillsets
- Examples
- Repository structure
- Compatibility or requirements
- Validation or test commands
- Roadmap
- Contributing
- License

## Quality Checks

- Commands are current
- Links resolve locally or externally
- Setup path is not host-locked unless intended
- Examples are concrete
- Claims are supported
- Advanced details are linked instead of front-loaded
- Roadmap is realistic and scoped

## Common Fixes

- Move long background below quickstart.
- Replace generic value claims with use cases.
- Add verification command after install.
- Split host-specific setup into separate docs.
- Add a troubleshooting link near setup.
