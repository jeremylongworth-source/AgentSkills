# Short-Form Hooks

## Core Workflow

1. Identify audience, platform, topic, creator voice, claim, desired action, and
   proof available.
2. Generate hook angles across curiosity, tension, mistake, transformation,
   contrarian, story, proof, and utility.
3. Keep hooks truthful and aligned with the actual content payoff.
4. Pair each hook with the first beat, visual idea, and retention risk.
5. Remove hooks that overpromise, shame, mislead, or create unsupported claims.
6. Recommend tests or variants when performance data is available.

## Safety Rules

- Do not promise virality, income, health, legal, financial, or guaranteed
  outcomes.
- Do not create clickbait that the content cannot satisfy.
- Do not fabricate personal stories, credentials, statistics, or testimonials.

## Deliverable Shape

For hook packs, provide:

- Audience and topic
- Hook variants
- Angle used
- First visual or beat
- Content payoff
- Risk or claim check
- Test recommendation
