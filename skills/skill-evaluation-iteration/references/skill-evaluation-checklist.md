# Skill Evaluation Checklist

- Scenario is realistic and specific.
- Baseline and skill-assisted outputs are comparable.
- Evaluation rubric is written before judging.
- Skill changes address observed gaps, not theoretical preferences.
- Frontmatter trigger remains precise.
- `agents/openai.yaml` display name, short description, and default prompt match the current skill.
- Account-level routing mentions the skill when it should be used globally.
- SKILL.md stays concise.
- Reference files hold deeper checklists or examples.
- No scaffold placeholders, TODOs, stale examples, or unrelated boilerplate remain.
- Official validator passes after edits.
