# Supply Chain Review Checklist

## Source

- Repository or package:
- Publisher:
- Commit or version:
- License:
- Release notes:
- Install command:

## Review

- Dependencies pinned
- Remote URLs explainable
- Scripts inspectable
- No opaque binaries unless justified
- No install-time network surprises
- No typosquatting or lookalike package names
- No broad host permissions
- Update path documented

## Recommendation

- Pin to commit or release.
- Fork before installing.
- Remove unneeded dependencies.
- Replace remote script fetches.
- Block until source is inspectable.
- Re-review after updates.
