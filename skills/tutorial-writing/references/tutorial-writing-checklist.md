# Tutorial Writing Checklist

## Tutorial Shape

- One learner
- One concrete outcome
- Clear prerequisites
- Copyable steps
- Expected output
- Verification after important steps
- Recovery notes
- Links to reference docs

## Avoid

- Multiple goals in one tutorial
- Long conceptual background before action
- Unverified commands
- Hidden costs
- Secret values
- Reference-manual sprawl

## Completion

The learner should end with:

- a working local state
- a generated artifact
- a completed task
- a clear next step
