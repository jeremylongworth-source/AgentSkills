# Write Spec Checklist

## Required Sections

- Problem statement
- Target users and segments
- Evidence summary
- Assumptions and open questions
- Goals and non-goals
- Requirements and constraints
- User flow or workflow
- Edge cases and failure states
- Acceptance criteria
- Measurement and validation plan
- Launch, support, and release notes

## Quality Checks

- Requirements are clear enough for engineering planning.
- Acceptance criteria are observable.
- Evidence is separated from opinion.
- Non-goals reduce scope creep.
- Dependencies and owner decisions are explicit.
- Analytics and QA needs are included when relevant.
